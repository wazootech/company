# Wazoo.dev AI Workforce

This repository defines the autonomous, protocol-driven AI workforce for the Wazoo ecosystem. It transitions from unstructured development toward a framework of **Protocol-Driven Generation**, where every agent's behavior and tone is governed by structured manifests.

## Core Architecture

The organization is structured around four functional pillars found in the `skills/` directory:

1.  **Software Engineering**: End-to-end feature delivery and system design.
2.  **Product Management**: Backlog prioritization and user research.
3.  **Technical Writing**: Documentation architecture and storytelling.
4.  **Community Management**: External signal translation and GTM strategy.

## Key Principles

Documentation and development follow a set of strict standards to ensure consistency and quality across the organization.

### Technical Writing Standards

All documentation must lead with context and identify a specific primary audience before execution. Writers favor root-relative paths, active voice, and Prose-First formatting. For more details, see the [technical-writing](skills/technical-writing/SKILL.md) manifest.

### Problem-First Prioritization

Product and engineering decisions are grounded in direct user research and support feedback. The workforce maintains a high signal-to-noise ratio by ignoring marketing language and focusing on developer "superpowers."

## Workflow

Initialize the workforce by pointing your orchestrator to the `skills/` directory. Each manifest defines the identity, security protocols (such as critical injection defense), and task management phases required for autonomous operation.
